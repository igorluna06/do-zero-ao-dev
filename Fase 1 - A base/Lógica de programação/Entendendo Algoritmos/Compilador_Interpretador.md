### Como é feita a comunicação com o computador

Compilador x Interpretador

### Compilador

É um programa responsável por traduzir o código-fonte, escrito em uma linguagem de alto nível,
para a linguagem de máquina, que o computador consegue executar.

Vantagens

- O código já é traduzido para a linguagem de máquina, com isso, se obtém um tempo menor de execução
- A confiabilidade do código final, já que os processos do compilador são bem severos e conseguem
validar e otimizar muito bem o código

Desvantagens

- Você não pode mudar o código sem ter recompilar todo programa, mesmo com diversas otimizações,
o compilador acaba sofrendo com o seu tempo de finalização do código
- Outro problema que podemos ter é no suporte de várias versões de um programa que é gerado a partir
do mesmo compilador(bibliotecas desatualizadas)

### Interpretador

É um programa que executa o código-fonte diretamente, traduzindo e rodando as instruções linha por linha,
sem gerar um arquivo executável final antecipadamente

Vantagens

- São fáceis de usar, especialmente para iniciantes
- A execução é linha por linha, sendo assim fica mais fácil de encontrar um erro

Desvantagens

- O tempo de execução é muito grande, devido a forma de que se é traduzido
- Para ser executado deve-se ter um programa correspondente ao interpretador
- Menos seguro
